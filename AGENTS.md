# CODEBUDDY.md

This file provides guidance to CodeBuddy Code when working with code in this repository.

## Commands

- **Install:** `pnpm install --frozen-lockfile`
- **Dev (watch):** `pnpm dev`
- **Build:** `pnpm build`
- **Lint:** `pnpm lint` (oxlint)
- **Format check:** `pnpm format:check` (oxfmt)
- **Typecheck:** `pnpm typecheck` (tsgo --noEmit)
- **CI parity:** `pnpm format:check && pnpm lint && pnpm build && pnpm typecheck`
- **Regenerate meta:** `pnpm generate:meta` (after changing `package.json` contributes)
- **Package VSIX:** `pnpm pack`

No test framework is configured.

## Architecture

This is a VS Code extension that decorates "magic comments" (e.g. `@ts-ignore`, `// prettier-ignore`) with inline annotations and hover markdown.

### Entry and startup

`src/index.ts` uses `reactive-vscode`'s `defineExtension`. Startup order: `useCommands()` -> `useCustomMagicComments()` -> `useAnnotations()`.

### Core data flow (pattern matching)

`src/features/annotations.ts` runs the match pipeline inside `watchEffect`:

1. Read active editor + document text via reactive-vscode composables
2. Skip unsupported/excluded languages
3. Execute each enabled provider's regex globally over full text
4. `match[1]` is the decorated token — provider regexes **must** capture the visible token in the first capture group
5. Regexes **must** include the `g` flag; non-global regexes are silently skipped
6. Convert matches to `Range` + hover info -> render via `useEditorDecorations`

### Key modules

- **`src/config.ts`** — Merges built-in + user `customMagicComments`, filters by `disabledCategories`/`disabledNames`. Exports `enabledMagicComments` (computed) and `config` (reactive VS Code settings).
- **`src/meta.ts`** — Auto-generated by `vscode-ext-gen`. Do not hand-edit. Regenerate with `pnpm generate:meta` when `package.json` contributes change.
- **`src/providers/`** — Built-in comment definitions, one file per tool (e.g. `typescript.ts`, `vue.ts`).
- **`src/types/magicComment.ts`** — `MagicComment` interface: `name`, `patterns`, optional `category`, `description`, `url`.
- **`src/utils/define.ts`** — `defineMagicComment()` identity helper for type-safe provider definitions.

### Provider authoring

1. Create `src/providers/<tool>.ts` with `defineMagicComment(...)` calls
2. Each provider must have `name`, `patterns` (string/RegExp/array), and usually `category`
3. Optional `description` and `url` appear in hover markdown
4. Export an array (e.g. `typescriptProviders`), keep items sorted where `// @keep-sorted` appears
5. Wire into `src/providers/index.ts` imports and `builtInMagicComments` array

### Runtime model

Uses `reactive-vscode` composables (`watchEffect`, `useActiveTextEditor`, `useDocumentText`) instead of imperative VS Code event wiring. Bundling via `tsdown` with `vscode` marked as never bundled.

### Placeholder providers

Some files in `src/providers/` (e.g. `eslint.ts`, `c8.ts`, `istanbul.ts`) are placeholders — they are **not** imported in `src/providers/index.ts` and are not active.
